# レポート保存 Skill

作業の振り返り・設計文書・調査結果などを `docs/reports/` に保存する。

## 保存先ルール

```
docs/reports/{年}/{月}/{日}/{連番}-{タイトル}.md
```

例: `docs/reports/2026/04/18/01-cli-args-design.md`

- **年月日**: 今日の日付を使う（`YYYY/MM/DD`、ゼロ埋め）
- **連番**: その日のフォルダ内の既存ファイル数 + 1（ゼロ埋め2桁）
- **タイトル**: 英語ケバブケース、内容を簡潔に表す

### 種別サフィックス

タイトルの末尾に種別を付けて区別する（サブフォルダーは作らない）。

| 種別 | サフィックス | 例 |
|---|---|---|
| 計画書 | `-plan` | `03-feature-plan.md` |
| 設計書・仕様書 | `-spec` / `-design` | `01-config-spec.md` |
| 実装レポート | `-report` | `06-cli-impl-report.md` |
| リファレンス | `-reference` | `01-cli-reference.md` |
| 手順書 | `-guide` / `-procedure` | `04-setup-guide.md` |
| ステータス報告 | `-status` | `08-project-status.md` |

## 手順

1. 今日のフォルダを確認: `ls docs/reports/YYYY/MM/DD/`
2. 既存ファイル数から次の連番を決定
3. フォルダがなければ作成: `mkdir -p docs/reports/YYYY/MM/DD`
4. ファイルを作成

## デフォルト保存先のルール

**保存先が明示されていない場合、常に `docs/reports/` を使う。**

- 「ここに保存して」→ `docs/reports/` に保存
- 「文書を作って」「表を作って」「まとめて」など、保存先の言及がない場合も `docs/reports/` に保存
- `docs/` ルート直下への保存は、ユーザーが明示的にパスを指定した場合のみ

## トリガー

| ユーザーの表現 | 動作 |
|---|---|
| "振り返り文書を作成して" | 直前の作業内容をレポートにまとめて保存 |
| "レポート書いて" | 同上 |
| "設計文書を保存" | 設計内容をレポートにまとめて保存 |
| "まとめを書いて" | 作業のまとめをレポートにまとめて保存 |
| "文書を保存して" | 作成・編集した .md ファイルを保存 |
| "ここに保存して" | 直前に作成した文書を `docs/reports/` に保存 |
| "保存して" | 直前に作成・編集した .md ファイルを `docs/reports/` に保存 |

### 暗黙のトリガー

ユーザーが「保存」「文書」「ドキュメント」「md」「まとめ」などのキーワードを含む指示をした場合、
保存先が明示されていなくても `docs/reports/` を保存先とみなして発動する。

## レポートのフォーマット

```markdown
---
generated_by: Claude {現在のモデル名}
date: {YYYY-MM-DD}
type: {report | spec | plan | guide | status | reference}
project: ai-to-web
---

# タイトル

> Generated by Claude {現在のモデル名} — {YYYY}年{M}月{D}日

## 背景
（なぜこの作業が必要だったか）

## 方針
（どういうアプローチを取ったか）

## 変更ファイル一覧
| # | ファイル | 変更 |
|---|---------|------|
| 1 | `path/to/file` | 変更内容 |

## 実装の詳細
（コード例や設計の説明）

## デバッグで判明した問題と修正
（あれば）

## 教訓
（今後に活かせるポイント）

---

> この文書は Claude {現在のモデル名} が生成しました。内容の正確性は人間のレビューを経て確認してください。
>
> Claude {現在のモデル名} — Anthropic
```

### ヘッダー（YAML frontmatter）— 必須

| フィールド | 値 |
|-----------|-----|
| `generated_by` | `Claude {現在のモデル名}`（例: `Claude Opus 4.7`） |
| `date` | `YYYY-MM-DD` |
| `type` | `report`, `spec`, `plan`, `guide`, `status`, `reference` のいずれか |
| `project` | `ai-to-web` |

### フッター（署名ブロック）— 必須

- `---` で本文と区切る
- 1行目: 免責（「人間のレビューを経て確認してください」）
- 2行目: 空の引用行（`>`）
- 3行目: モデル署名 `Claude {現在のモデル名} — Anthropic`

### その他のルール

- セクションは内容に応じて取捨選択する（全部必須ではない）
- コード例は実際の変更から抜粋
- 本文は日本語、コード・識別子は英語
