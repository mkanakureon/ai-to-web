**扱う概念**: 型幅（width）, バイト数, 値の範囲（0〜255, 0〜65535, ...）

**前提レッスン**: L0-3

| ペイン | 内容 |

| CpuBusMemory | メモリ領域に、a=0x41 が 1 セル、b=0x00 0x01 が 2 セル、c=0x00 0x00 0x01 0x00 が 4 セル、と並ぶ様子 |
| ExplainQuiz | 「型が違うと占めるバイト数が違う」という核 |

### ステップ構成

1. **`uint8_t a = 65`** を表示。Representation に `a: 65 (1 byte)`、メモリに 1 セル `0x41`
2. **`uint16_t b = 256`** を追加。Representation に `b: 256 (2 bytes)`、メモリに 2 セル。値が複数バイトにまたがることを観察

4. 値の**範囲**を表示（0〜255 / 0〜65535 / 0〜4294967295）。「何ビットあるか = 何通り表現できるか」
5. **Quiz**: uint16_t は何バイト？

## L2: エンディアン

**到達目標**: Little Endian と Big Endian の違いを、同じ 2 バイト値がメモリでどう並ぶかで説明できる。

**扱う概念**: バイト順, x86/ARM = Little Endian, ネットワーク = Big Endian

**前提レッスン**: L1

### 各ペインの観察対象

| ペイン | 内容 |
|-------|-----|
| Target | `uint16_t n = 0x1234;` |
| Representation | 値 `0x1234` / Big Endian 表記 `[0x12][0x34]` / Little Endian 表記 `[0x34][0x12]` |
| CpuBusMemory | メモリに 0x34, 0x12 の順で 2 セル（x86 = LE）。アドレス低位に下位バイト |
| ExplainQuiz | 「低いアドレスに下位バイト = Little Endian」 |

### ステップ構成

1. `uint16_t n = 0x1234;` を表示。Representation に値
2. **Big Endian** として並べると： `[高位:0x12][低位:0x34]` を Representation に
3. **Little Endian** として並べると： `[低位:0x34][高位:0x12]`。メモリに反映
4. 「x86/ARM は LE、ネットワークプロトコル（HTTP 含む）は BE」と注記
5. **Quiz**: x86 系のエンディアンは？

### Quiz

```
Q: x86 系 CPU のメモリ上のバイト順は?
   a) Big Endian（高位バイトが低アドレス）
   b) Little Endian（低位バイトが低アドレス）
   c) 設定で切替
正解: b
```

## L4: 配列とポインタ

**到達目標**: 配列アクセス `arr[i]` が「ベースアドレス + i * 要素サイズ」のメモリ読み取りだと観察できる。

**扱う概念**: 配列の先頭アドレス, 要素サイズ, インデックス計算

**前提レッスン**: L3

### 各ペインの観察対象

| ペイン | 内容 |
|-------|-----|
| Target | `uint32_t arr[3] = {10, 20, 30}; arr[1]` |
| Representation | `&arr[0] = 0x1000`, `&arr[1] = 0x1004`, `&arr[2] = 0x1008`（4 バイト刻み） |
| CpuBusMemory | `arr[1]` アクセス時、MAR=0x1004 → Address Bus → 該当セル選択 → Data Bus → MDR=20 |
| ExplainQuiz | 「配列アクセス = アドレス計算 + 読み取り」 |

### ステップ構成

1. `uint32_t arr[3] = {10, 20, 30};` を表示。メモリに 3 要素 × 4 バイト = 12 セル
2. Representation に各要素のアドレスを 0x1000, 0x1004, 0x1008 と明示
3. `arr[1]` を読みたい → CPU は 0x1000 + 1 × 4 = 0x1004 を計算
4. MAR に 0x1004 を入れ、L0-2 の読み取り手順で値 20 を MDR へ
5. **Quiz**: arr[2] のアドレスは？

### Quiz

```
Q: `uint32_t arr[3]` があり、&arr[0] = 0x1000 のとき、&arr[2] は?

   b) 0x1008

正解: b
```

## L5: ポインタの正体

**到達目標**: ポインタが「アドレスを保持する変数」であり、`*p` がそのアドレスからメモリを読む動作だと説明できる。

**扱う概念**: ポインタ変数, アドレス, デリファレンス (`*p`), アドレス取得 (`&x`)

**前提レッスン**: L4

### 各ペインの観察対象

| ペイン | 内容 |
|-------|-----|
| Target | `uint32_t x = 42; uint32_t *p = &x;` |
| Representation | `x` の値 `42` / `x` のアドレス `0x1000` / `p` の値 `0x1000` / `*p` の値 `42` |
| CpuBusMemory | メモリに x=42 が 4 バイト。別領域 (0x2000) に p=0x1000 が 4 バイト（値は番地） |
| ExplainQuiz | 「p の中身は番地。`*p` とすると番地を辿って読む」 |

### ステップ構成

1. `uint32_t x = 42;` を表示。x はメモリ 0x1000 に置かれる
2. `uint32_t *p = &x;` を追加。p は別のメモリ 0x2000 に置かれ、**値は 0x1000**
3. p の「値 = アドレス」だと強調。Representation に p の中身が `0x1000`
4. `*p` = p が指す先のメモリを読む → 0x1000 → 42
5. **Quiz**: ポインタは何を保持する？

### Quiz

```
Q: `uint32_t *p = &x;` のとき、p に入っているのは?
   a) x の値のコピー
   b) x の置かれたアドレス
   c) 関数のポインタ
正解: b
```

**前提レッスン**: L6

| ExplainQuiz | 「CPU が 4 バイト境界で int を読みたいので、間を空ける」 |

### ステップ構成

### Quiz
